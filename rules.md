# Rules

## General Rules
- Keep code simple and readable
- Must work in p5.js web editor
- No external libraries

## Coding Rules
- Use clear variable names
- Avoid overly complex logic
- Test small parts before adding more

## Game Rules
- Player moves left and right only
- Player shoots upward only
- Game ends when a ball hits the ground

## Development Rules
- Build core gameplay first
- Do not add visuals before mechanics work
- Add difficulty only after basic game works

## Object Use
- Use objects/classes for balls and bullets when helpful

## Restrictions
- Do not use features that break p5 editor
- Do not add unnecessary files
